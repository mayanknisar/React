1.1 Introduction & Setup ▶

What is Go?

Go (Golang) is a statically typed, compiled language designed at Google by Robert Griesemer, Rob Pike, and Ken Thompson. Released in 2009, it was built to address pain points in large-scale software systems: slow compile times, complex dependency management, and lack of built-in concurrency.

Why Go?

  • Fast compilation — large codebases compile in seconds
  • Garbage collected — no manual memory management
  • Concurrency built-in — goroutines & channels as first-class citizens
  • Single binary output — deploy anywhere, no runtime needed
  • Strong standard library — HTTP servers, JSON, crypto, and more out of the box
  • Enforced formatting — gofmt eliminates style debates

1.4 Functions ▶

Basic functions

func add(a, b int) int {
    return a + b
}

// Multiple return values (Go's killer feature)
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("cannot divide by zero")
    }
    return a / b, nil
}

// Named return values
func minMax(nums []int) (min, max int) {
    min, max = nums[0], nums[0]
    for _, v := range nums {
        if v < min { min = v }
        if v > max { max = v }
    }
    return // naked return
}

Variadic functions

func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

sum(1, 2, 3)           // 6
nums := []int{1,2,3}
sum(nums...)           // spread slice

First-class functions & closures

// Functions as values
apply := func(f func(int) int, x int) int {
    return f(x)
}

// Closure — captures surrounding scope
func counter() func() int {
    n := 0
    return func() int {
        n++
        return n
    }
}
c := counter()
fmt.Println(c(), c(), c()) // 1 2 3

defer

func readFile(path string) {
    f, _ := os.Open(path)
    defer f.Close()   // runs when readFile returns
    // ... work with f
}
💡Defers execute LIFO (last-in, first-out). Multiple defers stack — the last one registered runs first. Essential for resource cleanup (files, locks, connections).
1.5 Arrays, Slices & Maps ▶

Arrays (fixed size)

var a [3]int             // [0 0 0]

c := [...]int{4, 5, 6} // compiler counts length
fmt.Println(len(b))     // 3

Slices (dynamic — use these always)

s := []int{1, 2, 3}

// append — may allocate new backing array
s = append(s, 4, 5)

// make — preallocate
s2 := make([]int, 5, 10) // len=5, cap=10

// Slicing
a := []int{0,1,2,3,4}
fmt.Println(a[1:3])  // [1 2]  (low:high, high excluded)
fmt.Println(a[:3])   // [0 1 2]
fmt.Println(a[2:])   // [2 3 4]

// copy
dst := make([]int, len(a))
copy(dst, a)

Maps

// Create
m := map[string]int{
    "alice": 92,
    "bob":   85,
}

// Read, with ok-idiom
score, ok := m["alice"]
if !ok {
    fmt.Println("not found")
}

// Mutate
m["carol"] = 78
delete(m, "bob")

// Iterate (order is random!)
for k, v := range m {
    fmt.Printf("%s → %d\n", k, v)
}
💡Slices and maps are reference types — they point to an underlying data structure. Passing them to a function passes the reference, so the function can mutate the original.

2.1 Methods & Interfaces ▶

Methods on types

type Rectangle struct {
    Width, Height float64
}

// Value receiver (read-only)
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

// Pointer receiver (can mutate)
func (r *Rectangle) Scale(factor float64) {
    r.Width *= factor
    r.Height *= factor
}

Interfaces — implicit satisfaction

type Shape interface {
    Area() float64
    Perimeter() float64
}

// Rectangle satisfies Shape if it has both methods

    fmt.Printf("Area=%.2f Perimeter=%.2f\n",
        s.Area(), s.Perimeter())
}

// Empty interface — accepts any value
func describe(i interface{}) {
    fmt.Printf("%T: %v\n", i, i)
}
// Modern style (Go 1.18+): any
func describe(i any) { ... }

Type assertions & type switches

// Assertion with safety
var i interface{} = "hello"
s, ok := i.(string)

// Type switch

case int:
    fmt.Println("int:", v)
case string:
    fmt.Println("string:", v)
default:

}

2.2 Error Handling ▶

The error interface

// error is just an interface:
// type error interface { Error() string }

func openFile(path string) ([]byte, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        return nil, fmt.Errorf("openFile: %w", err)
    }
    return data, nil
}

Custom error types

type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation: %s — %s",
        e.Field, e.Message)
}

errors.Is / errors.As / wrapping

import "errors"

var ErrNotFound = errors.New("not found")

err := fmt.Errorf("db query: %w", ErrNotFound)

// Check error chain
if errors.Is(err, ErrNotFound) {
    fmt.Println("handle not-found case")
}

// Extract a specific error type
var ve *ValidationError
if errors.As(err, &ve) {
    fmt.Println("field:", ve.Field)
}

panic & recover

func safeDivide(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("recovered: %v", r)
        }
    }()
    return a / b, nil // panics if b==0
}

2.5 Testing in Go ▶

Writing tests

// math_test.go
package math

import "testing"

func TestAdd(t *testing.T) {
    got := Add(2, 3)
    want := 5
    if got != want {
        t.Errorf("Add(2,3) = %d; want %d", got, want)
    }
}

// Table-driven tests (idiomatic Go)
func TestDivide(t *testing.T) {
    cases := []struct{
        a, b  float64
        want  float64
        wantErr bool
    }{
        {10, 2, 5, false},
        {10, 0, 0, true},
    }
    for _, tc := range cases {
        got, err := Divide(tc.a, tc.b)
        if (err != nil) != tc.wantErr {
            t.Errorf("unexpected error: %v", err)
        }
        if got != tc.want { t.Errorf(...) }
    }
}

$ go test -cover          # coverage report
$ go test -bench=.        # benchmarks

Type parameters

// Generic function
func Map[T, U any](s []T, f func(T) U) []U {
    result := make([]U, len(s))
    for i, v := range s {
        result[i] = f(v)
    }
    return result
}

Constraints

import "golang.org/x/exp/constraints"

func Min[T constraints.Ordered](a, b T) T {
    if a < b { return a }
    return b
}

Min(3, 5)           // 3 (int)
Min("apple", "banana") // "apple" (string)

// Custom constraint
type Number interface {
    ~int | ~float64
}
func Sum[T Number](nums []T) T { ... }

Generic data structures

type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(item T) {
    s.items = append(s.items, item)
}

func (s *Stack[T]) Pop() (T, bool) {
    var zero T
    if len(s.items) == 0 { return zero, false }
    n := len(s.items) - 1
    item := s.items[n]
    s.items = s.items[:n]
    return item, true
}

3.1 Goroutines & Channels ▶

Goroutines — lightweight threads

func worker(id int) {
    fmt.Printf("worker %d starting\n", id)
    time.Sleep(time.Second)
    fmt.Printf("worker %d done\n", id)
}

func main() {
    for i := 1; i <= 5; i++ {
        go worker(i)    // starts goroutine
    }
    time.Sleep(2 * time.Second) // wait (use sync in practice)
}

Channels — goroutine communication

// Unbuffered (synchronous)
ch := make(chan int)
go func() { ch <- 42 }()
val := <-ch

// Buffered (async up to capacity)
bch := make(chan string, 3)
bch <- "first"
bch <- "second"

// Range over channel
go func() {
    for i := 0; i < 5; i++ { ch <- i }
    close(ch)     // signals end
}()
for v := range ch { fmt.Println(v) }

// Directional channels
func producer(ch chan<- int) { ch <- 1 }
func consumer(ch <-chan int) { v := <-ch; _ = v }

select statement

select {
case msg := <-ch1:
    fmt.Println("from ch1:", msg)
case msg := <-ch2:
    fmt.Println("from ch2:", msg)
case ch3 <- result:
    fmt.Println("sent to ch3")
case <-time.After(1 * time.Second):
    fmt.Println("timeout!")
default:
    fmt.Println("non-blocking fallthrough")
}
💡Go can run millions of goroutines concurrently — each starts at ~2KB of stack. The Go scheduler multiplexes them onto OS threads (GOMAXPROCS controls parallelism, defaults to CPU count).
3.2 Sync Package & Concurrency Patterns ▶

WaitGroup

var wg sync.WaitGroup

for i := 0; i < 5; i++ {
    wg.Add(1)
    go func(id int) {
        defer wg.Done()
        doWork(id)
    }(i)
}
wg.Wait()  // blocks until all Done()

Mutex

type SafeCounter struct {
    mu sync.Mutex
    v  map[string]int
}

func (c *SafeCounter) Inc(key string) {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.v[key]++
}

// RWMutex for read-heavy workloads
var rw sync.RWMutex
rw.RLock()   // multiple readers OK
rw.RUnlock()
rw.Lock()    // exclusive write
rw.Unlock()

Worker pool pattern

func workerPool(jobs <-chan int, results chan<- int, wg *sync.WaitGroup) {
    defer wg.Done()
    for j := range jobs {
        results <- j * j  // process job
    }
}

jobs    := make(chan int, 100)
results := make(chan int, 100)

var wg sync.WaitGroup
for w := 0; w < 5; w++ {  // 5 workers
    wg.Add(1)
    go workerPool(jobs, results, &wg)
}
for j := 1; j <= 50; j++ { jobs <- j }
close(jobs)
wg.Wait()
close(results)

sync.Once & sync.Map

var once sync.Once
var instance *DB

func GetDB() *DB {
    once.Do(func() { instance = connectDB() })
    return instance
}

// sync.Map: concurrent-safe map
var cache sync.Map
cache.Store("key", "value")
val, ok := cache.Load("key")
3.3 Context Package ▶

Why context?

The context package carries cancellation signals, deadlines, and request-scoped values across API boundaries and goroutines. It's the standard way to propagate cancellation in Go.

Creating contexts

ctx := context.Background()   // root, never cancelled
ctx := context.TODO()          // placeholder during refactor

// With cancellation
ctx, cancel := context.WithCancel(context.Background())
defer cancel()

// With timeout
ctx, cancel := context.WithTimeout(ctx, 5*time.Second)
defer cancel()

// With deadline
deadline := time.Now().Add(30 * time.Minute)
ctx, cancel := context.WithDeadline(ctx, deadline)

Using context in goroutines

func fetchData(ctx context.Context, url string) ([]byte, error) {
    req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
    resp, err := http.DefaultClient.Do(req)
    if err != nil { return nil, err }
    defer resp.Body.Close()
    return io.ReadAll(resp.Body)
}

// Goroutine that respects cancellation
func doWork(ctx context.Context) {
    for {
        select {
        case <-ctx.Done():
            fmt.Println("cancelled:", ctx.Err())
            return
        default:
            processNextItem()
        }
    }
}

Context values

type ctxKey string

ctx = context.WithValue(ctx, ctxKey("userID"), "u-123")

userID := ctx.Value(ctxKey("userID")).(string)
💡Pass context as the first argument to every function that does I/O. Never store context in a struct. Cancelling a parent context cancels all its children.
3.4 Building a REST API ▶

net/http with custom mux

type Server struct {
    router *http.ServeMux
    store  Store
}

func NewServer(store Store) *Server {
    s := &Server{router: http.NewServeMux(), store: store}
    s.router.HandleFunc("GET /api/users", s.listUsers)
    s.router.HandleFunc("POST /api/users", s.createUser)
    s.router.HandleFunc("GET /api/users/{id}", s.getUser)
    return s
}

func (s *Server) createUser(w http.ResponseWriter, r *http.Request) {
    var req CreateUserRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }
    user, err := s.store.CreateUser(r.Context(), req)
    if err != nil {
        http.Error(w, "internal error", http.StatusInternalServerError)
        return
    }
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusCreated)
    json.NewEncoder(w).Encode(user)
}

// Path values (Go 1.22+)
func (s *Server) getUser(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id")   // extracts {id}
    ...
}

// Middleware pattern
func logging(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        defer func() {
            log.Printf("%s %s %v", r.Method, r.URL.Path, time.Since(start))
        }()
        next.ServeHTTP(w, r)
    })
}

Go memory model essentials

  • Variables escape to the heap when their lifetime exceeds the function — the compiler decides (escape analysis)
  • The GC is tri-color concurrent mark-and-sweep — since Go 1.14 STW pauses are under 1ms
  • Stack grows dynamically (starts ~8KB, copies on overflow)
  • Avoid allocations in hot paths: reuse buffers with sync.Pool

sync.Pool — buffer reuse

    New: func() any { return new(bytes.Buffer) },
}

buf := bufPool.Get().(*bytes.Buffer)

main.go wiring

func main() {
    cfg := loadConfig()

    db, err := connectDB(cfg.DatabaseURL)
    if err != nil { log.Fatal(err) }
    defer db.Close()

    store  := store.New(db)
    svc    := service.New(store)
    server := handler.NewServer(svc)

    srv := &http.Server{
        Addr:         ":" + cfg.Port,
        Handler:      server,
        ReadTimeout:  10 * time.Second,
        WriteTimeout: 30 * time.Second,
        IdleTimeout:  60 * time.Second,
    }

    // Graceful shutdown
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)

    go func() {
        if err := srv.ListenAndServe(); err != http.ErrServerClosed {
            log.Fatal(err)
        }
    }()

    <-quit
    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()
    srv.Shutdown(ctx)
    log.Println("server stopped gracefully")
}
